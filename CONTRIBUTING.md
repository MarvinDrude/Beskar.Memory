# Contributing to Beskar.Networking

Thank you for your interest in improving Beskar.CodeGeneration! We welcome community contributions to help make this tool more robust and efficient.

## Our Approach to Contributions

While we encourage and appreciate Pull Requests (PRs), please note the following policy:

1. **Right of Refusal:** The maintainer reserves the right to accept, modify, or reject any contribution for any reason.
2. **Design Alignment:** Contributions must align with the architectural goals and coding standards of the project.
3. **Ownership:** By submitting a Pull Request, you agree that the maintainer may modify your code and that the final implementation is subject to the maintainer's discretion.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please open an issue describing the problem, including:
* Your environment (OS, .NET version).
* Steps to reproduce the issue.
* Expected vs. actual behavior.

### 2. Feature Requests
Please open an issue to discuss new features before starting any work. This ensures the proposed changes fit the project's roadmap.

### 3. Submitting Pull Requests
To submit a change:
1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/YourFeature`).
3. Ensure your code follows the project's coding standards.
4. Add or update unit tests as necessary.
5. Submit your PR with a clear description of the changes.

## Coding Standards

To maintain consistency throughout the codebase, please adhere to the following:

* **Language:** Use C# as the primary programming language.
* **Style:** Follow standard .NET naming conventions (PascalCase for classes/methods).
* **Indentation:** Use **3 spaces** for indentation instead of 4 or tabs.
* **Comments:** Provide XML documentation comments for public members in English.
* **Clean Code:** Keep methods small and focused on a single responsibility.
