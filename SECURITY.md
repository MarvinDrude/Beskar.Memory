# Security Policy

## Supported Versions

We are actively trying to provide security updates for newer versions of the project.

## Reporting a Vulnerability

We take the security of our code generation tools seriously. If you discover a potential security vulnerability, please do not disclose it publicly via GitHub issues. Instead, follow this process:

1. **Private Report:** Send a detailed description of the vulnerability to the maintainer.
2. **Details to Include:**
    * A description of the issue.
    * Potential impact.
    * Steps to reproduce or a Proof of Concept (PoC).
3. **Response Time:** You can expect an acknowledgment of your report within 48–72 hours.

## Our Approach to Security Changes

As with all contributions to this project:
* The maintainer reserves the **exclusive right** to determine if a reported issue constitutes a security vulnerability.
* We reserve the right to accept, reject, or modify any proposed security patches or PRs.
* Any security-related PRs submitted without prior private discussion may be closed to prevent public disclosure of the exploit.

## Best Practices for Users

When using **Beskar.Markdown** in your C# projects, we recommend:
* Running the generator within a controlled environment.
* Regularly updating to the latest version to receive the most recent security patches.
* Reviewing generated code before deployment to ensure it meets your organization's security standards.

## Code Integrity

To maintain the security of the codebase, please ensure all contributions follow our coding standard:
* **Indentation:** 3 spaces.
* **Language:** C# (English comments).
